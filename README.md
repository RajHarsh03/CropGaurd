# рҹҢҝ CropGuard вҖ“ AI-Powered Crop Disease Detection

An end-to-end deep learning application that detects plant diseases from leaf images and provides detailed remedy suggestions.

## рҹҺҜ Features

- рҹ“ё Upload leaf images via drag-and-drop or file picker  
- рҹӨ– AI disease prediction using MobileNetV2 transfer learning  
- рҹ“Ҡ Confidence score with top-5 prediction breakdown  
- рҹ’Ҡ Disease description, symptoms, causes, and remedies  
- рҹӣЎпёҸ Prevention tips for each condition  
- вҡЎ Real-time inference via FastAPI REST API  
- рҹ“ұ Responsive modern UI (React + Tailwind CSS)  

## рҹ§¬ Supported Classes (15 total)

| Plant | Conditions |
|-------|-----------|
| Tomato | Bacterial Spot, Early Blight, Late Blight, Leaf Mold, Septoria Leaf Spot, Spider Mites, Target Spot, Yellow Leaf Curl Virus, Mosaic Virus, Healthy |
| Potato | Early Blight, Late Blight, Healthy |
| Bell Pepper | Bacterial Spot, Healthy |

## рҹҡҖ Quick Start

### Prerequisites
- Python 3.10+
- Node.js 18+
- npm 9+

---

### Step 1 вҖ“ Clone the repository

```bash
git clone https://github.com/yourusername/crop-disease-detection.git
cd crop-disease-detection
```

---

### Step 2 вҖ“ Set up the Dataset

Place your PlantVillage dataset at:
```
crop-disease-detection/Dataset/PlantVillage/
```

The folder must contain subdirectories named after each class:
```
Dataset/PlantVillage/
  Tomato_Bacterial_spot/
  Tomato_Early_blight/
  Tomato_Late_blight/
  Tomato_Leaf_Mold/
  Tomato_Septoria_leaf_spot/
  Tomato_Spider_mites_Two_spotted_spider_mite/
  Tomato__Target_Spot/
  Tomato__Tomato_YellowLeaf__Curl_Virus/
  Tomato__Tomato_mosaic_virus/
  Tomato_healthy/
  Potato___Early_blight/
  Potato___Late_blight/
  Potato___healthy/
  Pepper__bell___Bacterial_spot/
  Pepper__bell___healthy/
```

---

### Step 3 вҖ“ Train the ML model

```bash
cd ml

# Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate

# Install ML dependencies
pip install -r requirements.txt

# Run training (GPU recommended; CPU works but is slow)
python train.py
```

This will:
- Train MobileNetV2 with transfer learning (two phases: head + fine-tune)
- Save the model to `backend/trained_model/crop_disease_model.keras`
- Save class names to `backend/trained_model/class_names.json`
- Generate training accuracy/loss plots in `ml/outputs/`

Training time: ~20вҖ“40 min on GPU, ~2вҖ“4 hours on CPU.

---

### Step 4 вҖ“ Start the Backend

```bash
cd backend

# Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Copy environment file
cp .env.example .env

# Start the server
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

API will be available at: http://localhost:8000  
Interactive docs: http://localhost:8000/docs

---

### Step 5 вҖ“ Start the Frontend

```bash
cd frontend

# Install dependencies
npm install

# Copy environment file
cp .env.example .env

# Start development server
npm run dev
```

App will be available at: http://localhost:5173

## рҹ§  Model Architecture

```
Input (224Г—224Г—3)
  в””в”Җв”Җ MobileNetV2 (ImageNet pretrained, frozen)
        в””в”Җв”Җ GlobalAveragePooling2D
              в””в”Җв”Җ BatchNormalization
                    в””в”Җв”Җ Dense(512, relu) + Dropout(0.4)
                          в””в”Җв”Җ Dense(256, relu) + Dropout(0.3)
                                в””в”Җв”Җ Dense(15, softmax)
```

**Training strategy:**
1. Phase 1 (head training): Freeze base, train classifier head for 10 epochs at lr=1e-3
2. Phase 2 (fine-tuning): Unfreeze top layers (after layer 100), train for 15 epochs at lr=1e-5

### Step 1 вҖ“ Deploy via Render Blueprint

1. Push your repo to GitHub
2. Go to [render.com](https://render.com) вҶ’ Sign in with GitHub
3. Click **"New"** вҶ’ **"Blueprint"**
4. Select your repository
5. Render auto-detects the `render.yaml` and creates both services
6. Click **"Apply"**

The `render.yaml` defines two services:

**Backend (Web Service):**
```yaml
- type: web
  name: cropguard-backend
  runtime: python
  plan: free
  buildCommand: cd backend && pip install -r requirements.txt
  startCommand: cd backend && uvicorn app.main:app --host 0.0.0.0 --port $PORT
  healthCheckPath: /api/v1/health
```

**Frontend (Static Site):**
```yaml
- type: static_site
  name: cropguard-frontend
  rootDir: frontend
  buildCommand: npm install && npm run build
  staticPublishPath: dist
```

---

### Step 2 вҖ“ Configure Environment Variables

After both services are created, note their URLs from the Render dashboard (e.g., `https://cropguard-backend-xxxx.onrender.com` and `https://cropguard-frontend-xxxx.onrender.com`).

**Backend environment variables** (Service вҶ’ Environment):

| Key | Value |
|-----|-------|
| `PYTHONUNBUFFERED` | `1` |
| `MODEL_PATH` | `./trained_model/crop_disease_model.keras` |
| `CORS_ORIGINS` | `https://cropguard-frontend-xxxx.onrender.com` |
| `ENVIRONMENT` | `production` |

**Frontend environment variables** (Service вҶ’ Environment):

| Key | Value |
|-----|-------|
| `VITE_API_BASE_URL` | `https://cropguard-backend-xxxx.onrender.com` |

> вҡ пёҸ **Important:** The env var must be `VITE_API_BASE_URL` (not `VITE_API_URL`). This must match what the frontend code reads in `src/services/api.js`.

---

### Step 3 вҖ“ Redeploy Both Services

After updating environment variables:

1. **Backend:** Go to the backend service вҶ’ **Manual Deploy** вҶ’ **Deploy latest commit**
2. **Frontend:** Go to the frontend service вҶ’ **Manual Deploy** вҶ’ **Clear build cache & deploy**

> рҹ’Ў The frontend **must** be rebuilt (not just redeployed) because `VITE_*` environment variables are baked into the JavaScript bundle at build time.

### How CORS Works

The backend reads the `CORS_ORIGINS` environment variable at startup and adds those origins to the allowed list (see `backend/app/core/config.py`). Local development origins (`localhost:5173`, `localhost:3000`) are included by default.

You do **not** need to edit `backend/app/main.py` for CORS вҖ” just set the `CORS_ORIGINS` env var in Render with your frontend URL.

### Troubleshooting

**вқҢ "Publish directory dist does not exist!" on frontend deploy?**
- Ensure `rootDir` is set to `frontend` in Render dashboard (Settings вҶ’ Root Directory)
- Publish Directory should be `dist` (not `frontend/dist`)
- The `vite.config.js` must have `build.outDir: "dist"` set explicitly

**вқҢ Backend crashes / model not loading?**
- Verify `backend/trained_model/crop_disease_model.keras` is committed to Git (check `.gitignore`)
- Check Render logs: Service вҶ’ Logs
- Ensure `MODEL_PATH` env var is set to `./trained_model/crop_disease_model.keras`

**вқҢ "Backend server is offline" on frontend?**
- Verify the backend is running: visit `https://your-backend.onrender.com/api/v1/health`
- Ensure `VITE_API_BASE_URL` (not `VITE_API_URL`) is set correctly in frontend env vars
- **Rebuild** the frontend after changing env vars (Clear build cache & deploy)

**вқҢ "Cannot reach the server" when clicking Analyze?**
- Check CORS: `CORS_ORIGINS` on the backend must include the exact frontend URL
- Check browser DevTools вҶ’ Network tab for CORS errors
- Ensure the backend start command uses `--port $PORT` (not a hardcoded port)

**вқҢ Cold start too slow?**
- Render free tier sleeps after 15 mins of inactivity
- First request after sleep takes ~30вҖ“60 seconds (TensorFlow model loading)
- Upgrade to paid tier for always-on production use
